# История версий KayFit

Все значимые изменения фиксируются здесь в обратном хронологическом порядке.

Формат: `[версия] — дата`. Секции: **Добавлено**, **Исправлено**, **Изменено**, **Удалено**.

---

## [1.0.3+1] — 2026-05-17

> Тег: `v1.0.3+1` · Патч-релиз: исправление iOS auth session bug (пользователей разлогинивало после закрытия приложения).

### Исправлено

**iOS сессия — пользователь больше не разлогинивается при закрытии приложения**
- Провайдер авторизации теперь `keepAlive` — Riverpod не уничтожает состояние сессии при навигации по экранам
- При возврате приложения из фона сессия тихо обновляется без показа экрана входа
- Сетевой таймаут или недоступность сервера больше не приводит к разлогину — токены остаются в Keychain
- Keychain недоступен при фоновом запуске (до разблокировки устройства) — теперь не трактуется как «нет токенов»
- Устранена гонка состояний: глобальный logout-callback убран, маршрутизация теперь реактивная через state

### Тесты
- Добавлено 20 регресс-тестов: роутер, перехватчик 401, AI consent, Settings logout
- Добавлены тесты жизненного цикла приложения (AppLifecycleState.resumed)
- Итого: 244 теста, все зелёные

## [1.0.2+1] — 2026-05-12

> Тег: `v1.0.2+1-sent` · Отправлено разработчикам для сборки и загрузки в App Store Connect.

### Добавлено

**KayFit 2.0 — полный редизайн интерфейса**
- **Дневник V2**: лента календаря (неделя + месяц), кольца КБЖУ в стиле Apple Activity, карточки приёмов пищи, удаление свайпом, кольца статуса (зелёный/красный относительно цели)
- **Чат V2**: пузыри с шагами обработки AI, панель вложений (камера / голос / штрихкод), значок добавления еды (MealAddedBadge), чип «Скорректировать» после AI-ответа с едой, баннер-disclaimer со ссылками на ВОЗ/USDA
- **Распознавание V2**: полноэкранная камера → экран «Распознаю» → карточка результата в новом дизайне
- **Настройки V2**: навигация кнопкой «Назад», дизайн-токены KF2, нижняя панель убирается при активном флаге KF2_JOURNAL
- Шрифты Geist (интерфейс) и JetBrains Mono (числа) встроены в приложение

**BodyForm — выбор формы тела**
- Новый шаг в онбординге: пользователь выбирает текущую и желаемую форму тела (шкала 0–6, отдельные силуэты для мужчин и женщин)
- Приложение автоматически рассчитывает целевой вес через формулу сухой мышечной массы
- Целевой вес передаётся в `/api/calculate` при цели `lose_weight`
- Доступ после онбординга: Настройки → BodyForm

**CI / Android**
- Добавлен GitHub Actions workflow: при каждом push в master собирается debug-APK для Android и публикуется как артефакт на 7 дней

### Исправлено

**Чат**
- Фото в чате больше не зависало без ответа — исправлена «тихая» ошибка при добавлении фото
- После добавления еды AI теперь отправляет коучинговое сообщение с оценкой и советом

**Локализация**
- Подписи макронутриентов (БЕЛКИ / ЖИ / УГЛЕВОДЫ / Готово) в карточке еды теперь переключаются вместе с языком приложения
- Снэкбар при отклонении AI consent теперь учитывает язык (был всегда на английском)
- Экран документов: Политика конфиденциальности и Пользовательское соглашение на русском теперь отображаются корректно (ранее подставлялась английская версия)
- Все упоминания «Carb Counter» в документах и строках Bluetooth заменены на «Kayfit»

**App Store**
- Дисклеймер с источниками (ВОЗ/USDA) добавлен на экран Дневника V2 — требование Guideline 1.4.1

### Изменено

- Акцентный цвет унифицирован: Apple system blue `#007AFF` применён везде — в устаревшем интерфейсе и KF2
- Источник питательных веществ FatSecret убран из интерфейса (требование Guideline 5.2.1): бейдж `FatSecret` → `Nutrition DB`, метка `FATSECRET` → `DB`

### Удалено

- Зависимость `app_links ^6.1.0` удалена — тянула устаревший iOS-символ, deep-link в этой версии не используется
- `NSSpeechRecognitionUsageDescription` удалён из `Info.plist` — API распознавания речи не используется
- Иконка iPad (`AppIcon76x76@2x~ipad.png`) удалена — приложение только для iPhone

## [1.0.0+10] — 2026-05-07

> Тег: *(не проставлен)* · Первая сборка, отправленная в App Store Connect. Получила реджект 1 апреля.

### Добавлено

- Первый публичный релиз: дневник питания, распознавание еды по фото и голосу, AI-чат, онбординг с расчётом КБЖУ
- Поддержка русского языка — переключение в Настройках
- PrivacyInfo.xcprivacy — манифест конфиденциальности по требованию Apple
- Разрешения на камеру, микрофон, фотобиблиотеку и Bluetooth вынесены в локализованные `InfoPlist.strings` (EN + RU)
- Токены авторизации перенесены в iOS Keychain

### Исправлено (в ходе подготовки к релизу)

- Онбординг: цветовая палитра переведена с розовой (OBColors) на синюю (AppColors `#007AFF`) — все экраны подхватили автоматически
- Weight Step: убран ручной ввод целевого веса; значение берётся только из расчёта BodyForm. При целях `muscle_gain` / `maintain` целевой вес не задаётся, график не строится
- Login: добавлена синяя тема, ссылки на ToS стали кликабельными
- AI Consent: при отклонении пользователь остаётся в приложении (ручной ввод); выход из аккаунта убран — требование App Store H-5
- Skip-диалог в онбординге: убрано жёлтое подчёркивание текста (наследование от Material 3 DefaultTextStyle)
- iOS Deployment Target исправлен с 13.0 на 16.0 во всех конфигурациях `project.pbxproj` — несоответствие с Podfile вызывало linker warnings
- `Pods-Runner-frameworks.sh`: добавлена `local source=""` перед if/elif в `install_framework()` — исправлен `unbound variable` при деплое на физическое устройство

### Изменено

- Google Sign-In удалён (отсутствовал CLIENT_ID в `GoogleService-Info.plist`)
- `webview_flutter` удалён (нигде не использовался)
- Экран подписки: кнопка «View plans» заменена на метку «Free» (роут `/tariffs` не существовал)
- Debug-баннер скрыт (`debugShowCheckedModeBanner: false`)
- Приложение собрано только для iPhone (`TARGETED_DEVICE_FAMILY = 1`)
