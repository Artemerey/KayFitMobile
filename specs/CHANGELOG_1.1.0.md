# Changelog — KayFit 1.1.0+1

> Previous release: 1.0.3+1

## New Features

### RevenueCat IAP — subscription gate
- Интегрирован `purchases_flutter: ^8.0.0` (RevenueCat) для управления подписками
- Subscription gate: платный контент закрыт для пользователей без активной подписки
- Paywall экран с оформлением подписки, кнопкой восстановления и условиями
- Кнопка ввода промо-кода (`promo code redemption`) на экране paywall

### Locale-aware paywall — EN/USD поддержка
- Paywall автоматически переключается между RU (₽) и EN ($) на основе локали устройства
- Backend subscription gate: сервер проверяет статус подписки через RevenueCat webhook

### Фоновое распознавание + push-уведомление
- Фото еды можно отправить из чата — распознавание идёт в фоне (без блокировки UI)
- После завершения распознавания приходит push-уведомление с результатом

### Клавиатура, голос и быстрое редактирование в чате/журнале
- Чат автоскроллится при открытии клавиатуры
- Голосовой ввод подхватывает вес (fallback: если Whisper не распознал число — запрашивает повторно)
- Копирование блюда в другой день из журнала
- Inline редактирование отложенных (pending) позиций прямо в чате

### Inline вес-пилюля и быстрое редактирование в журнале
- Вес порции теперь редактируется прямо в строке журнала (без перехода на отдельный экран)
- Fast-path edit: для простых изменений не открывается полный экран редактирования

### Способ оплаты (UI-stub)
- Добавлен экран выбора способа оплаты: карта и СБП (пока UI-заглушка без логики оплаты)

### Onboarding — "У меня уже есть аккаунт"
- Кнопка "У меня уже есть аккаунт" теперь доступна с первого экрана онбординга (была только на определённых шагах)

## Bug Fixes

### Роутер — показ онбординга после переустановки
- Исправлен баг: после удаления и переустановки приложения Keychain сохранял auth-токен, поэтому пользователь попадал сразу в главный экран минуя онбординг. Теперь при переустановке (SharedPreferences сброшены, Keychain жив) выполняется сброс сессии и показывается онбординг.

### Распознавание фото — compressWithFile → compressWithList
- Заменён метод сжатия изображений: `compressWithFile` падал на некоторых форматах. Теперь используется `compressWithList` + добавлен `mounted` guard перед операциями с `BuildContext`.

### Таймауты на AI/chat эндпоинтах
- `receiveTimeout` на Claude-backed эндпоинтах и `/api/chat/send` увеличен до 120 секунд (был слишком мал для медленных ответов LLM)

### Правильное сохранение веса для прошлых дат
- Редактирование веса блюда в журнале за прошедшую дату теперь корректно сохраняется (баг: данные не персистировались при dateOffset ≠ 0)
